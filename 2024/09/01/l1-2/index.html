文組生的資訊素養：第二課　關於電腦的那些事-1
Sep 1, 2024

今天來講一些電腦硬體相關的事好了，可能算是我的簡易版計概，計概真的太大本了，又塞滿一堆古代內容，我就寫我覺得比較重要的部分就好。說是這麼說啦…我去翻了一下以前做的簡報，又發現我其實講很細…以前覺得重要的點又跟現在不太一樣，滿頭痛的。

所以硬體相關的應該會有不只一篇，因為真的太多東西可以講了，那我盡量不要講太深，還是以一種引起興趣的方式來介紹給大家囉！

位元 Bit

上次講完二進制不知道大家吸收的如何，有大概提到過位元（bit）就是二進制中的一位，但幾乎都不會只有一位嘛，只有一位的話就只能表示 true 跟 false ，但我們還會需要計算更多東西，所以用到更多位元的話，就會用到其他單位來計算。

首先就是位元組（byte），8個 bit 就是1個 byte。從這邊開始就可以慢慢開始熟悉更多跟二進制有關的概念了，因為電腦本身用的方式就是二進制嘛，所以各式各樣的單位，其實都會用2的n次方來定義。

那以歷史意義來說，8位元除了長得工整，可以用兩位十六進制表示以外，最主要會用8位元當作一個基本單位其實是因為早期電腦系統都是以英文（更準確來說就是拉丁字母）為主，所使用的編碼集是ASCII（讀作/ˈæski/ ，啊嘶ㄎㄧˋ），總共有128個字元，其實是7位元，但7位元實在是不太漂亮，所以索性就用8位元當作一個位元組，多出來的一個位元可以用來做一些其他的功能。

位元組是基本單位，那更大的單位如何表示呢？這裡開始會出現大家熟悉的名詞了，或著說應該至少看過前三個…？

以上，基本上不用記得啦…

日常生活中會用到的就前四個， kB 、 MB 、 GB 跟 TB 而已，最多最多可能看到 PB 吧，某些新聞或技術文章會出現。其餘應該暫時還用不到，就像課本裡那些京、恆河沙、不可思議之類的單位，就算真的有這麼大的數通常也是直接用數量級表示了，不太會用單位。

所以也就是說其實很簡單啦，這邊用到的這些單位其實就是各差1000（103）倍而已，但是只有這一種十進制的單位嗎？其實也不只…以kB為例，跟他長得很像的有這些：

十進制 二進制
kB kbit(kb) KiB Kibit(Kib)
kilobyte kilobit kibibyte kibibit
103 bytes 103 bits 210 byte 210 bits

情況一瞬間變得非常糟糕，好多奇怪的單位出現了。不要緊張，我們從看過的東西一個一個來。

先以十進制這邊來說，可以發現除了前面講過的大寫 B 以外，出現了另一種小寫 bit 或 b ，沒錯！小寫 b 的版本就是位元的數量級單位，翻成中文的話 kB 就是千位元組， kbit 或 kb 就是千位元。所以這兩個其實還算好理解吧！

二進制這邊就是另一個概念了，每個數量級都會相差210倍，也就是1024倍。老樣子，是因為電腦實際上的設計方式都是以2的倍數為基礎，例如匯流排寬度、儲存單位的大小等等，都會是以2的倍數來設計。所以才會有二進制的數量級單位，否則用十進制表示的話數字會變很醜，也不方便。

那問題來了，請問我們平常用的到底是什麼制的單位呢？

很遺憾，這裡的答案大概是不一定…相信聰明的你一定有發現一件很矛盾的事情，我們日常生活中幾乎不會見到 KiB 、 MiB 這些單位，但似乎它們的定義又更合理一點，因為既然是電腦內的計量單位就應該要盡量符合電腦的設計才對吧？

其實這兩者的混用是一直以來都存在的問題，對我來說這裡面有不少因素。先講好的例子好了，也是我少數誇獎Apple的時候了，只能說不愧是賈伯斯，（還是聲明一下，我真的不熟macOS，這邊是網路上找到的資料，有錯還請多多包涵），在2009年8月28日發布的Mac OS X Snow Leopard中，包含了一項更新，讓這之後在MacOS上的檔案大小皆是以十進制單位來表示的。

十進制單位是國際電工委員會（IEC）推薦的定義，也算是公制單位的一種（例如公斤、公升、公尺，公制單位都以十進制定義，方便計算），所以基本上這應該是要全球統一的標準，但大家也知道我們現在還是經常使用一些奇奇怪怪的單位，但二進制字首的單位似乎又沒這麼奇怪…

沒錯喔，因為電腦硬體實際上都是以二進制為基礎設計，所以像是記憶體大小，因為實體儲存位址的關係，最直觀的方式就是用二進制數量級來表示，也因此Windows到現在也還是使用二進制的單位來表示檔案的大小。

也因此有另一個單位就出現了，雖然它比較不是被訂定出來的單位，卻也非常廣泛的在使用，那就是大寫K的 KB （只有K有區別，其他字首就是混用），等於是 KiB 的縮寫，如果你是用Windows系統的話現在可以打開檔案總管看看，應該會看到單位是 KB 的檔案。

那實際上十進制跟二進制單位的不同會造成什麼影響呢？比方說我們買硬碟可能會看到一張這樣混亂的規格…
